# List

The `List` component in **Kontra TUI** is a vertical layout container that stacks components top-to-bottom - like a column - but **respects each child's preferred height**. This makes it ideal for elements with varying size, like menus, multi-line text, and mixed content.

---

## Basic Usage

You can create a list by passing any number of components to the constructor:

<CodeBlock
  border
  lang="cpp"
  code={`
    auto list = std::make_shared<List>(
        item1,
        item2,
        item3
    );`}
/>

Use `.set_gap(int)` to insert empty lines between components:

<CodeBlock
  border
  lang="cpp"
  code={`
    list->set_gap(1);
    `}
/>

---

## Features & Behavior

The `List` is different from a `Flex(Column)` layout because:

- It **does not stretch** children to fill available space.
- It respects each component's actual height.
- It's ideal for **text blocks**, **menus**, **forms**, or **dynamic content**.

You can include any component inside a list:

<CodeBlock
  border
  lang="cpp"
  code={`
    auto input = std::make_shared<InputBox>();
    input->set_text("Even InputBoxes work!");
    auto list = std::make_shared<List>(
        item1,
        item2,
        input
    );`}
/>

---

## Full Layout Example

Here's how you can build a list with various content:

<CodeBlock border 
  lang="cpp"
  code={`
    auto item1 = std::make_shared<Text>("This is a single-line item.");
    
    auto item2 = std::make_shared<Text>("This is a\\nmulti-line item\\nthat takes up more space.");
    
    auto input_item = std::make_shared<InputBox>();
    
    input_item->set_text("Lists can contain other components too!");

    auto main_list = std::make_shared<List>(
        item1,
        item2,
        input_item
    );

    main_list->set_gap(1);`}

/>

---

## Render It on Screen

Wrap the `List` in a `Screen` and optionally a `Border`-

<CodeBlock border 
  lang="cpp"
  code={`
    auto screen = std::make_shared<Screen>(
        std::make_shared<Border>(main_list)
    );

    kontra::run(screen, [&](char ch) {
        if (ch == 17) exit(0); // Ctrl+Q to quit
        input_item->handle_input(ch); // Pass input to InputBox
    });`}

/>

## TL;DR

Use `List` when you want to vertically stack components of varying height - like messages, comments, or mixed content blocks. It's a flexible layout that doesn't force equal sizing and works great with dynamic or reactive elements like `InputBox`, `Text`, and even nested lists.
