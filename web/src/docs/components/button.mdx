# Button

The `Button` component is used to create interactive elements in your Kontra TUI app. Buttons can trigger custom actions when "clicked" and respond to focus states for keyboard interaction.

## üõ†Ô∏è Creating a Button

A basic button can be created like this:

<CodeBlock
  border
  lang="cpp"
  code={`
    auto button = std::make_shared<Button>("Click Me!", []() {
        // Code to run when clicked
    });
`}
/>

You can also customize its appearance using the `ButtonStyleBuilder`.

---

## Styling Buttons

The `ButtonStyleBuilder` makes it easy to define visual appearance for both normal and active (focused) states.

<CodeBlock
  border
  lang="cpp"
  code={`
    auto style = ButtonStyleBuilder()
    .set_color(ansi::FG_WHITE)
    .set_background_color(ansi::BG_BLUE)
    .set_color_active(ansi::FG_BLUE)
    .set_background_color_active(ansi::BG_BRIGHT_WHITE)
    .set_bold(true)
    .build();`}
/>

- `set_color(...)`- normal text color
- `set_background_color(...)`- normal background
- `set_color_active(...)`- focused/active text color
- `set_background_color_active(...)`- focused/active background
- `set_bold(...)`- makes the button text bold

All color options come from [`ansi`](/docs/ansi).

You can pass this style when creating a button:

<CodeBlock
  border
  lang="cpp"
  code={`
    auto button = std::make_shared<Button>("Styled", []() {
        std::cout << "Clicked!";
    }, style);`}
/>

## Managing Focus

Since Kontra doesn't manage focus automatically, you need to set which button is active manually.

<CodeBlock border 
  lang="cpp"
  code={`
    button1->set_active(true);
    button2->set_active(false);

    std::shared_ptr<Button> active_button = button1;
    `}

/>

To switch focus on `Tab`-

<CodeBlock
  border
  lang="cpp"
  code={`
    if (ch == '\\t') {
        // Toggle focus
        button1->set_active(!button1->is_active());
        button2->set_active(!button2->is_active());
        active_button = button1->is_active() ? button1 : button2;
    }`}
/>

---

## Handling Click

To simulate a button click (e.g., when user presses `Enter`):

<CodeBlock
  border
  lang="cpp"
  code={`
    if (ch == '\\n' || ch == '\\r') {
        active_button->click();
    }`}
/>

## Complete Layout Example

You can stack buttons in a layout like this:

<CodeBlock border 
  lang="cpp"
  code={`
    auto layout = std::make_shared<List>(
        display_text,
        button1,
        button2
    );
    layout->set_gap(1);

    auto screen = std::make_shared<Screen>(std::make_shared<Border>(layout));`}

/>

## TL;DR

- Use `Button` to make parts of your terminal interface **interactive**.
- Each button has a **label**, an **onClick callback**, and **customizable styles**.
- Handle **keyboard input** manually (`Tab`, `Enter`) to manage focus and clicks.
- Combine `Button` with `Text`, `List`, and `Flex` to build responsive **interactive layouts**.
