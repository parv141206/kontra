# Button

The `Button` component lets you make your terminal UI interactive â€” click, tab, or mouse-press your way through clean TUI controls.  
Use it to perform actions, show feedback, or simply to give your app some life.

---

## Creating a Button

You can create a button with just a label and an `onClick` callback:

<CodeBlock
  border
  lang="cpp"
  code={`
    auto button = std::make_shared<Button>("Click Me!", []() {
      // Do something here
    });`}
/>

## Styling Buttons

Style your buttons with ease â€” including hover/focus state!

<CodeBlock
  border
  lang="cpp"
  code={`
    // 1. Create a style for the button when it's NOT focused (inactive)
    auto inactive = StyleBuilder()
        .set_color(ansi::FG_WHITE)
        .set_background_color(ansi::BG_BLUE)
        .set_bold(true)
        .build();

    // 2. Create a style for when it IS focused (active)
    auto active = StyleBuilder()
        .set_color(ansi::FG_BLUE)
        .set_background_color(ansi::BG_BRIGHT_WHITE)
        .set_bold(true)
        .build();

    // 3. Combine them using the ButtonStyleBuilder
    auto button_style = ButtonStyleBuilder()
        .set_inactive_style(inactive)
        .set_active_style(active)
        .build();`}
/>

- `set_inactive_style(...)` - sets the full style for the normal state.
- `set_active_style(...)` - sets the full style for the focused/hover state.

Each method takes a complete style object created with `StyleBuilder` (from `text.hpp`).

Now apply the style like this:

<CodeBlock
  border
  lang="cpp"
  code={`
    auto button = std::make_shared<Button>("Styled", []() {
      // ... click logic ...
    }, button_style);`}
/>

## Handling Focus + Input

Kontra gives _you_ full control over what's active and clickable.

### Switch focus with `Tab`:

<CodeBlock
  border
  lang="cpp"
  code={`
    if (event.key == '\\t') {
      active_button->set_active(false); // Deactivate the old one
      if (active_button == button1) {
          active_button = button2;
      } else {
          active_button = button1;
      }
      active_button->set_active(true); // Activate the new one
    }`}
/>

### Trigger click with `Enter`:

<CodeBlock
  border
  lang="cpp"
  code={`
    if (event.key == '\\n' || event.key == '\\r') {
      active_button->click();
    }`}
/>

## Mouse Support (Yup, we got that)

You can even respond to mouse clicks:

<CodeBlock
  border
  lang="cpp"
  code={`
    if (event.type == EventType::MOUSE_PRESS) {
      for (const auto& btn : buttons) {
          if (btn->contains(event.mouse_x, event.mouse_y)) {
              btn->click();
              // Also update the focus to the clicked button
              active_button->set_active(false);
              btn->set_active(true);
              active_button = btn;
              break; // Stop after handling one click
          }
      }
    }`}
/>

## Complete Layout Example

<CodeBlock border 
  lang="cpp" 
  code={`
    // A Text component to show feedback
    auto display_text = std::make_shared<Text>("Click or Tab to test!");

    // Define a shared style for the buttons
    auto btn_inactive_style = StyleBuilder().set_bold(true).build();
    auto btn_active_style = StyleBuilder().set_bold(true).set_background_color(ansi::BG_BRIGHT_BLACK).build();
    auto button_style = ButtonStyleBuilder()
        .set_inactive_style(btn_inactive_style)
        .set_active_style(btn_active_style)
        .build();

    auto button1 = std::make_shared<Button>("Set Message 1", [&]() {
      // The text component must be captured by reference to be modified
    }, button_style);
    
    // We'll create a dynamic Text component for the message
    auto message_display = std::make_shared<Text>([&](){ return "Waiting..."; });

    // Now define the button actions
    button1->set_on_click_callback([&](){ *message_display = Text("Button 1 Clicked!"); });

    auto button2 = std::make_shared<Button>("Set Message 2", [&]() {
      *message_display = Text("Button 2 Clicked!");
    }, button_style);


    // Manage focus state
    button1->set_active(true);
    std::shared_ptr<Button> active_button = button1;
    std::vector<std::shared_ptr<Button>> buttons = {button1, button2};

    auto layout = std::make_shared<List>(message_display, button1, button2);
    layout->set_gap(1);

    auto screen = std::make_shared<Screen>(std::make_shared<Border>(layout));

    kontra::run(screen, [&](const InputEvent& event) {
      if (event.type == EventType::KEY_PRESS) {
        if (event.key == '\\t') {
          // Swap focus
          active_button->set_active(false);
          active_button = (active_button == button1) ? button2 : button1;
          active_button->set_active(true);
        } else if (event.key == '\\n' || event.key == '\\r') {
          active_button->click();
        }
      } else if (event.type == EventType::MOUSE_PRESS) {
          for (auto& btn : buttons) {
            if (btn->contains(event.mouse_x, event.mouse_y)) {
              btn->click();
              active_button->set_active(false);
              btn->set_active(true);
              active_button = btn;
              break;
            }
        }
      }
    });
    `} />

## TL;DR

- Use `Button` to make parts of your terminal interface **interactive**.
- Each button has a **label**, an **onClick callback**, and **customizable styles**.
- Handle **keyboard input** manually (`Tab`, `Enter`) to manage focus and clicks.
- Combine `Button` with `Text`, `List`, and `Flex` to build responsive **interactive layouts**.
